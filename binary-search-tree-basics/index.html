Binary search tree basics

A binary search tree (BST) is a tree where all elements in the left sub-tree (if any) are less than the root and all elements in the right sub-tree (if any) are bigger than the root, and the sub-trees are themselves BST. All values should be unique in BST because they are usually used as keys to retrieve dictionary (or map) values.

Recursive tree traversal

  1. In-order traversal: recursively traverse the left sub-tree, then visit the root, then recursively traverse the right sub-tree.
  2. Pre-order traversal: the same with in-order traversal, but visit the root first.
  3. Post-order traversal: the same with in-order traversal, but visit the root last.

Binary search

Start with the root, if it is not the target, compare the target with the root and decide which sub-tree to go next. Go all the way down to the leaf of the tree and stop if the target is never found.

Insert

  1. Find the place the value should be by searching the value in the tree and go all the way down to the bottom of the tree (current_node == null). If the value is found along the way, we probably shouldn’t insert it again.
  2. Make the new node and link it to its parent. To do this, remember to keep track of the parent of the current_node when we do the search in step 1.

Max/Min

To find the maximum value of a BST (or any sub-tree of it), start with the root, and go all the way to the right until there is no more right child. Similar procedure for finding minimum value.

Successor/Predecessor

To find the smallest node bigger than a given node, there are two cases:

a. node has a right child:

The successor is the minimum value of the sub-tree rooted at the right child, so call min(node.right_child) to find it.

b. node has no right child:

The succesor is the parent of node if it is a left child of the parent. Otherwise, to one-level up and check whether the parent is a left child of the grand-parent, etc. Go all the way up to the top until we get a node who is a left child of its parent, and that parent is the successor of node. If we reach the root of the tree, node does not have a successor and it is the maximum node of the BST.

Similar procedure for the predecessor.

Delete

a. node has no child:

Just set the link from node‘s parent to node as null and delete the node.

b. node has one child:

Let’s say node has a left child but no right child. All we need is to link the left sub-tree of node with its parent and delete node.

c. node has two children but the right child of node does not have left child:

Put the right child in the place of node and delete node. And “old right child” become the parent of the left child.

Or, we can do the similar thing to the left child if the right child does has a left child.

d. node has two children and non of the above is valid:

  1. Find the successor of node, put it in the place of node and delete node.
  2. Put the successor’s right child B (if exists) in the place of the successor, and so now B is the left child of the seccesor’s parent A.
